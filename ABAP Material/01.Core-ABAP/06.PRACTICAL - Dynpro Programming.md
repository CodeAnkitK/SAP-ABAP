# ABAP Dynpro Corporate Training Demo

**3 Screens, Subscreen, Messages, Custom Table, and Navigation**

Classic dialog programming is still one of the best ways to teach core SAP ABAP screen concepts. This demo combines navigation, subscreens, menus, messages, and table handling in one compact example that new learners can study and practice.

The program is built as a **module pool** with three primary screens and one subscreen. It shows how a real SAP dialog application is structured, how user commands are processed, and how custom database tables are maintained through screen logic.

By the end of this demo, learners will understand:

## Custom Table Design

The demo uses a simple employee table so learners can understand database interaction clearly.

### Table: `ZTRN_EMP`

| Field | Key | Type | Purpose |
| :-- | :-- | :-- | :-- |
| MANDT | Yes | MANDT | Client |
| EMP_ID | Yes | NUMC 6 | Employee ID |
| EMP_NAME | No | CHAR 40 | Employee name |
| DEPT | No | CHAR 20 | Department |
| DESIGNATION | No | CHAR 30 | Designation |
| SALARY | No | CURR 13(2) | Salary |
| CURRENCY | No | WAERS | Currency |
| JOIN_DATE | No | DATS | Joining date |
| STATUS | No | CHAR 1 | Active/Inactive |

### Technical Settings

Use these settings for the table:

- Data class: `APPL0`
- Size category: `0` or `1`
- Buffering: not required for the demo

### Message Class: `ZMC_TRAINING`

| Number | Text |
| :-- | :-- |
| 001 | Employee ID is required |
| 002 | Employee Name is required |
| 003 | Record saved successfully for Employee ID \&1 |
| 004 | Record updated successfully for Employee ID \&1 |
| 005 | Record deleted successfully for Employee ID \&1 |
| 006 | Employee ID \&1 not found |
| 007 | Screen \&1 opened |
| 008 | Please enter valid salary |
| 009 | Popup subscreen displayed |
| 010 | Action cancelled by user |
| 011 | Navigation to screen \&1 completed |
| 012 | Confirm delete for employee \&1 |

## ABAP Source Code

Below is the full module pool source with detailed comments for learners.

```abap
PROGRAM zmp_training_demo MESSAGE-ID zmc_training.
" This declares the module pool and connects it to the message class.

TABLES: ztrn_emp.
" This work area is linked to the custom table for screen field handling.

DATA: ok_code        TYPE sy-ucomm,
      save_ok        TYPE sy-ucomm,
      gv_mode        TYPE c LENGTH 1,
      gv_subscreen   TYPE sy-dynnr VALUE '0150',
      gv_popup_on    TYPE abap_bool VALUE abap_false,
      gv_answer      TYPE c LENGTH 1.
" ok_code stores the function code from user action.
" save_ok preserves the command before ok_code is cleared.
" gv_mode can be used for future display/change/create logic.
" gv_subscreen stores the current subscreen number.
" gv_popup_on controls whether the subscreen area is shown.
" gv_answer holds the response from the delete confirmation popup.

DATA: gs_emp         TYPE ztrn_emp,
      gt_emp         TYPE STANDARD TABLE OF ztrn_emp,
      gv_emp_id      TYPE ztrn_emp-emp_id.
" gs_emp is the single employee work area.
" gt_emp holds multiple employee records.
" gv_emp_id is available for extra processing if needed.

CONTROLS: tc_emp TYPE TABLEVIEW USING SCREEN 0200.
" This defines the table control used on screen 0200.

************************************************************************
* SCREEN 0100
************************************************************************

MODULE status_0100 OUTPUT.
  " This module runs before screen 0100 is displayed.

  SET PF-STATUS 'STAT100'.
  " This activates the toolbar and menu for the screen.

  SET TITLEBAR 'TIT100'.
  " This sets the title bar of the screen.

  MESSAGE s007 WITH '0100'.
  " This informs the user that screen 0100 is open.
ENDMODULE.

MODULE user_command_0100 INPUT.
  " This module processes user actions on screen 0100.

  save_ok = ok_code.
  " Store the function code for processing.

  CLEAR ok_code.
  " Clear the command field to prevent repeated execution.

  CASE save_ok.
    WHEN 'NEXT1' OR 'SCR2'.
      " Navigate to the employee maintenance screen.
      MESSAGE s011 WITH '0200'.
      SET SCREEN 0200.

    WHEN 'SCR3'.
      " Navigate to the demo screen.
      MESSAGE s011 WITH '0300'.
      SET SCREEN 0300.

    WHEN 'POPON'.
      " Show the popup subscreen.
      gv_popup_on = abap_true.
      MESSAGE s009.

    WHEN 'POPOFF'.
      " Hide the popup subscreen.
      gv_popup_on = abap_false.
      MESSAGE s010.

    WHEN 'BACK' OR 'EXIT' OR 'CANC'.
      " Exit the program.
      LEAVE PROGRAM.

    WHEN OTHERS.
      " Default message for unhandled commands.
      MESSAGE i010.
  ENDCASE.
ENDMODULE.

************************************************************************
* SCREEN 0200
************************************************************************

MODULE status_0200 OUTPUT.
  " PBO module for screen 0200.

  SET PF-STATUS 'STAT200'.
  " This activates the screen's GUI status.

  SET TITLEBAR 'TIT200'.
  " This sets the screen title.

  PERFORM get_data_0200.
  " This loads data for display in the table control.
ENDMODULE.

MODULE user_command_0200 INPUT.
  " PAI module for screen 0200.

  save_ok = ok_code.
  " Store the function code.

  CLEAR ok_code.
  " Reset the command field.

  CASE save_ok.
    WHEN 'SCR1'.
      " Navigate to the home screen.
      MESSAGE s011 WITH '0100'.
      SET SCREEN 0100.

    WHEN 'SCR3'.
      " Navigate to the demo screen.
      MESSAGE s011 WITH '0300'.
      SET SCREEN 0300.

    WHEN 'GET'.
      " Read a record from the database.
      PERFORM get_single_employee.

    WHEN 'SAVE'.
      " Save or update the employee data.
      PERFORM save_employee.

    WHEN 'DEL'.
      " Delete the employee record.
      PERFORM delete_employee.

    WHEN 'CLEAR'.
      " Clear the work area for new entry.
      CLEAR gs_emp.
      MESSAGE i010.

    WHEN 'BACK'.
      " Return to the home screen.
      SET SCREEN 0100.

    WHEN 'EXIT' OR 'CANC'.
      " Leave the program completely.
      LEAVE PROGRAM.

    WHEN OTHERS.
      " Default message for invalid commands.
      MESSAGE i010.
  ENDCASE.
ENDMODULE.

************************************************************************
* SCREEN 0300
************************************************************************

MODULE status_0300 OUTPUT.
  " PBO module for screen 0300.

  SET PF-STATUS 'STAT300'.
  " Activate the toolbar and menu for this screen.

  SET TITLEBAR 'TIT300'.
  " Set the title text for the screen.

  IF gv_subscreen IS INITIAL.
    " Make sure a subscreen number is available.
    gv_subscreen = '0150'.
  ENDIF.
ENDMODULE.

MODULE user_command_0300 INPUT.
  " PAI module for screen 0300.

  save_ok = ok_code.
  " Store the function code.

  CLEAR ok_code.
  " Clear the command after reading it.

  CASE save_ok.
    WHEN 'SCR1'.
      " Navigate to the home screen.
      MESSAGE s011 WITH '0100'.
      SET SCREEN 0100.

    WHEN 'SCR2'.
      " Navigate to the employee screen.
      MESSAGE s011 WITH '0200'.
      SET SCREEN 0200.

    WHEN 'SUB150'.
      " Show the popup subscreen.
      gv_subscreen = '0150'.
      MESSAGE s009.

    WHEN 'SHOWS'.
      " Show a success message.
      MESSAGE s003 WITH gs_emp-emp_id.

    WHEN 'SHOWI'.
      " Show an information message.
      MESSAGE i007 WITH '0300'.

    WHEN 'SHOWW'.
      " Show a warning message.
      MESSAGE w010.

    WHEN 'SHOWE'.
      " Show an error message.
      MESSAGE e001.

    WHEN 'BACK'.
      " Return to the home screen.
      SET SCREEN 0100.

    WHEN 'EXIT' OR 'CANC'.
      " Exit the program.
      LEAVE PROGRAM.

    WHEN OTHERS.
      " Default fallback message.
      MESSAGE i010.
  ENDCASE.
ENDMODULE.

************************************************************************
* SUBSCREEN 0150
************************************************************************

MODULE status_0150 OUTPUT.
  " PBO for the subscreen.
  " This is kept simple for the demo.
ENDMODULE.

MODULE user_command_0150 INPUT.
  " PAI for the subscreen.
  " No additional action is required here.
ENDMODULE.

************************************************************************
* FORMS
************************************************************************

FORM get_data_0200.
  " Read employee records for display.

  SELECT *
    FROM ztrn_emp
    INTO TABLE gt_emp
    UP TO 50 ROWS.
ENDFORM.

FORM get_single_employee.
  " Fetch a single employee by ID.

  IF gs_emp-emp_id IS INITIAL.
    MESSAGE e001.
  ENDIF.

  SELECT SINGLE *
    FROM ztrn_emp
    INTO gs_emp
    WHERE emp_id = gs_emp-emp_id
      AND mandt  = sy-mandt.

  IF sy-subrc = 0.
    MESSAGE s007 WITH 'Employee fetched'.
  ELSE.
    MESSAGE e006 WITH gs_emp-emp_id.
  ENDIF.
ENDFORM.

FORM validate_employee.
  " Validate employee data before save.

  IF gs_emp-emp_id IS INITIAL.
    MESSAGE e001.
  ENDIF.

  IF gs_emp-emp_name IS INITIAL.
    MESSAGE e002.
  ENDIF.

  IF gs_emp-salary IS INITIAL OR gs_emp-salary < 0.
    MESSAGE e008.
  ENDIF.

  IF gs_emp-currency IS INITIAL.
    gs_emp-currency = 'INR'.
  ENDIF.

  IF gs_emp-status IS INITIAL.
    gs_emp-status = 'A'.
  ENDIF.
ENDFORM.

FORM save_employee.
  " Insert or update employee data.

  PERFORM validate_employee.

  gs_emp-mandt = sy-mandt.
  " Client field must always be filled before database update.

  SELECT SINGLE *
    FROM ztrn_emp
    INTO @DATA(ls_old)
    WHERE mandt  = @sy-mandt
      AND emp_id = @gs_emp-emp_id.
  " Check whether the record already exists.

  IF sy-subrc = 0.
    UPDATE ztrn_emp FROM gs_emp.
    IF sy-subrc = 0.
      MESSAGE s004 WITH gs_emp-emp_id.
    ELSE.
      MESSAGE e010.
    ENDIF.
  ELSE.
    INSERT ztrn_emp FROM gs_emp.
    IF sy-subrc = 0.
      MESSAGE s003 WITH gs_emp-emp_id.
    ELSE.
      MESSAGE e010.
    ENDIF.
  ENDIF.

  PERFORM get_data_0200.
  " Refresh table control data after save.
ENDFORM.

FORM delete_employee.
  " Delete the selected employee after confirmation.

  IF gs_emp-emp_id IS INITIAL.
    MESSAGE e001.
  ENDIF.

  CALL FUNCTION 'POPUP_TO_CONFIRM'
    EXPORTING
      titlebar              = 'Delete Confirmation'
      text_question         = 'Do you want to delete this employee?'
      text_button_1         = 'Yes'
      icon_button_1         = 'ICON_OKAY'
      text_button_2         = 'No'
      icon_button_2         = 'ICON_CANCEL'
      default_button        = '2'
      display_cancel_button = space
    IMPORTING
      answer                = gv_answer.
  " This popup asks the user to confirm deletion.

  IF gv_answer = '1'.
    DELETE FROM ztrn_emp
      WHERE mandt  = sy-mandt
        AND emp_id = gs_emp-emp_id.

    IF sy-subrc = 0.
      MESSAGE s005 WITH gs_emp-emp_id.
      CLEAR gs_emp.
      PERFORM get_data_0200.
    ELSE.
      MESSAGE e006 WITH gs_emp-emp_id.
    ENDIF.
  ELSE.
    MESSAGE i010.
  ENDIF.
ENDFORM.
```

## Screen Flow Logic

### Screen 0100

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0100.
  IF gv_popup_on = abap_true.
    CALL SUBSCREEN sub_pop INCLUDING sy-repid '0150'.
  ENDIF.

PROCESS AFTER INPUT.
  IF gv_popup_on = abap_true.
    CALL SUBSCREEN sub_pop.
  ENDIF.
  MODULE user_command_0100.
```


### Screen 0200

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0200.

PROCESS AFTER INPUT.
  MODULE user_command_0200.
```


### Screen 0300

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0300.
  CALL SUBSCREEN sub_demo INCLUDING sy-repid gv_subscreen.

PROCESS AFTER INPUT.
  CALL SUBSCREEN sub_demo.
  MODULE user_command_0300.
```


### Screen 0150

```abap
PROCESS BEFORE OUTPUT.
  MODULE status_0150.

PROCESS AFTER INPUT.
  MODULE user_command_0150.
```

## Suggested Build Steps

Follow this sequence while building the demo in SAP:

1. Create the transparent table in SE11.
2. Maintain technical settings and activate the table.
3. Create the message class in SE91.
4. Build the module pool program in SE80.
5. Create all four screens in SE51.
6. Define GUI statuses in SE41.
7. Assign title bars.
8. Create the transaction code in SE93.
9. Test all navigation paths and message buttons.

This sequence keeps the implementation simple and avoids missing dependencies.
