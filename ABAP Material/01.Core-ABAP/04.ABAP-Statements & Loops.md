# Core ABAP Statements & Dynamic Programming Concepts

## Overview

These concepts form the backbone of ABAP programming, enabling control flow, user interaction, data transfer, and efficient memory handling.

---

## Core Concepts

### Conditional & Looping Statements

Used to control program flow and process data.

- **IF / ELSE / CASE** → Decision-making based on conditions  
- **LOOP / DO / WHILE** → Repetitive execution of logic  
- Commonly used with internal tables  

**👉 Example Use Case:**  
Loop through sales data and display only high-value orders  

---

### Selection Screen Inputs

Used to capture user input dynamically.

- **Checkbox** → Multiple selections allowed  
- **Radio Button** → Only one option selectable  

**👉 Example Use Case:**  
User selects *Domestic* or *International* report view  

---

### Program & Data Handling

- **SUBMIT** → Call another report program  
- **EXPORT / IMPORT** → Share data via memory between programs  

**👉 Example Use Case:**  
One report triggers another and passes data for processing  

---

### Field-Symbols (Dynamic Programming)

- Acts like a pointer to memory (no data duplication)  
- Declared using `< >` (e.g., `<fs_data>`)  
- Enables dynamic data handling and improves performance  

**👉 Example Use Case:**  
Modify internal table rows dynamically without copying data  

---

## Practical Code Examples

### 1. Conditional Statements (IF / CASE)

```abap
DATA lv_amount TYPE i VALUE 500.

IF lv_amount > 300.
  WRITE: 'High Value Order'.
ELSE.
  WRITE: 'Low Value Order'.
ENDIF.

CASE lv_amount.
  WHEN 100.
    WRITE: 'Basic'.
  WHEN 500.
    WRITE: 'Standard'.
  WHEN OTHERS.
    WRITE: 'Other'.
ENDCASE.

```

### 2. Looping Statements (LOOP / DO)
```abap
DATA: lt_numbers TYPE TABLE OF i,
      lv_num TYPE i.

APPEND 10 TO lt_numbers.
APPEND 20 TO lt_numbers.

LOOP AT lt_numbers INTO lv_num.
  WRITE: / lv_num.
ENDLOOP.

DO 3 TIMES.
  WRITE: / 'Repeat'.
ENDDO.
```

### 3. Selection Screen (Checkbox & Radio Button)
```abap
PARAMETERS: p_chk AS CHECKBOX.
PARAMETERS: p_r1 RADIOBUTTON GROUP r1,
            p_r2 RADIOBUTTON GROUP r1.

IF p_chk = 'X'.
  WRITE: 'Checkbox Selected'.
ENDIF.
```

### 4. SUBMIT Statement

```abap
SUBMIT z_sample_report AND RETURN.
```

### 5. EXPORT / IMPORT (Memory Handling)
```abap
DATA lv_text TYPE string VALUE 'Hello'.

EXPORT lv_text TO MEMORY ID 'TEST'.

IMPORT lv_text FROM MEMORY ID 'TEST'.
WRITE: lv_text.
```

### 6. Field-Symbols (Dynamic Access)

```abap
DATA: lt_data TYPE TABLE OF i.

FIELD-SYMBOLS: <fs> TYPE i.

APPEND 100 TO lt_data.

LOOP AT lt_data ASSIGNING <fs>.
  <fs> = <fs> + 10.
  WRITE: / <fs>.
ENDLOOP.
```

## Key Insight

* **IF / CASE** → Decision-making
* **LOOP / DO** → Repetition
* **SUBMIT** → Program execution chaining
* **EXPORT / IMPORT** → Data sharing across programs
* **Field-Symbols** → High-performance dynamic handling

**👉 These are the building blocks of real ABAP programs.**
